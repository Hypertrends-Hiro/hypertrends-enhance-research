Guía de uso — Frontend

Plan API Payload universal (detalle) Guía Backend Mapa de eventos

El frontend no llama a Braze directamente. Envía el payload universal a POST /api/v1/telemetry/ingest (o el endpoint que definan). El servicio telemetry valida, deduplica y publica a Braze.

Qué debes enviar siempre

  • meta.message_id — UUIDv7 por evento (reutilizar en retry).
  • meta.occurred_at — new Date().toISOString() en el instante del evento.
  • meta.source_system — nombre fijo de tu app (ej. kwilthealth.com).
  • meta.source_channel — frontend.
  • event.name + event.time — normalmente mismo instante que occurred_at.
  • user.external_id o user.anonymous_id — al menos uno.

Qué debes enviar para UX / sesión

CampoRecomendación
session.session_idObligatorio para unir page views y clicks en una visita.
session.page_url, session.routeRecomendado en page_viewed y navegación.
event.propertiesSin PII innecesaria; preferir ids y flags.

Qué no hagas desde el frontend

  • No actualices perfil Braze directo (email, phone, custom attrs masivos) — eso va por backend o por ingest con user.attributes bajo política.
  • No envíes el mismo message_id para dos eventos distintos.
  • No mandes payloads “legacy” mezclados; usa siempre el envelope del plan API.
